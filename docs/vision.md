# microDify 产品愿景

> 面向人类维护者的产品背景文档。Agent 执行规范见根目录 `CLAUDE.md`，架构图谱见 `docs/microDify-map.md`。

## 定位

microDify 是 [Dify](https://github.com/langgenius/dify) 的简化版 AI Agent 平台，面向团队内部 20-50 人使用，Docker Compose 本地部署，一人开发维护。

核心原则：**模块化单体架构，只做高价值功能，砍掉一切不必要的复杂度。**

## 主要模块与功能（7 个核心模块）

| 模块 | 功能 | 简化策略 |
| -------------- | -------------------------------------------------------------- | -------------------------- |
| **模型接入** | OpenAI 协议 + Anthropic 协议，统一抽象接口，预留扩展 Adapter | 不做负载均衡、熔断、协议适配层 |
| **Agent 智能体** | ReAct 自主推理循环 + 动态工具调用，支持绑定知识库和预定义工具，SSE 流式返回推理过程 | 不做多 Agent 协作、自主发现工具、代码沙箱执行 |
| **知识库管理** | 支持 PDF / DOCX / TXT / Markdown / CSV 五种格式上传、语义分块、向量化存储 | 不做知识管道 DSL、不拆可编排节点 |
| **RAG 引擎** | pgvector 向量存储，BM25 + 向量混合检索，简单 Rerank | 只支持 pgvector，不接入 12 种向量库 |
| **对话应用** | 创建应用 → 绑定 Prompt 模板 → 可选关联知识库，SSE 流式对话 | 合并"知识库问答"和"AI 助手"两个场景 |
| **工作流引擎** | 可视化编排，5-6 种节点（开始、LLM、知识检索、条件判断、变量转换、结束），线性串联 + if/else 分支，变量传递 | 不做并行、循环、子工作流、工作流即工具 |
| **Prompt IDE** | 变量插值 `{{variable}}`，基础模板管理 | 不做多版本管理、A/B 测试、评分 |

## 支持的核心场景

- **知识库问答**：上传文档 → 自动分块嵌入 → 对话中检索相关内容并生成回答
- **AI 对话助手**：配置 Prompt 模板的通用聊天机器人（客服、写作、翻译等）
- **工作流自动化**：串联多个 AI 步骤 + 条件分支 + 变量传递的半自动化流程
- **Agent 智能体**：ReAct 推理循环 + 动态工具调用，自主规划完成复杂任务（信息检索、数据分析、外部 API 调用等）

## 明确不做（及原因）

| 不做 | 原因 |
| -------------------------- | ---------------------- |
| 工作流并行 / 循环 / 子工作流嵌套 | 中等复杂度场景不需要 |
| 工作流发布为可复用工具 | 工作流本身不支持嵌套 |
| 多 Agent 协作 / Agent 自主发现工具 | 复杂度指数级增长，单人无法驾驭 |
| 代码沙箱执行（E2B / Docker） | Agent 只调用预定义工具，不执行任意代码 |
| 知识管道 DSL | 固定管道覆盖 5 种格式足够 |
| 插件系统（SDK + 运行时 + 市场） | 一人无法维护生态 |
| 多租户 / 命名空间隔离 | 单一团队使用 |
| 企业安全合规（RBAC / 审计 / 加密） | 内部工具，TLS 由 Caddy 处理 |
| 多版本 Prompt 管理 / A/B 测试 | LLMOps 专业平台的事 |
| 模型运行时管理（负载均衡 / 熔断） | 两个 Adapter，直接调 SDK |
| Prometheus + Grafana 可观测体系 | 结构化日志 + 简单用量仪表盘足够 |

## 后期可能拓展

| 拓展 | 条件 |
| ----------------------------------------- | ------------------------ |
| 更多 LLM Provider（Ollama / DeepSeek / 通义千问） | 通过 Adapter 模式扩展，无需改动核心代码 |
| 工作流升级为完整 DAG（并行 / 循环） | 用户对条件分支熟悉后提出需求 |
| 多 Agent 协作 + 代码沙箱 | Agent 单智能体无法满足需求时 |
| 轻量插件 / Hook 机制 | 出现 3+ 个硬编码外部集成时 |
| 用量与成本仪表盘 | 团队开始关注 Token 消耗分配时 |
| SSO / OIDC 对接 | 团队 IT 要求统一认证时 |
